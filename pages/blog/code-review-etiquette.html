Code Review Etiquette

How to give feedback that improves code without destroying morale.

Code reviews are the number one source of conflict in engineering teams. A comment like "Why did you do it this way?" can ruin someone's day. It sounds accusatory, even if you didn't mean it.

Google's Engineering Guidelines explicitly state: " Critique the code, not the person." This sounds simple, but linguistic nuances make it hard.

The "Question" vs. The "Statement"

Statements feel like orders. Questions feel like collaboration. Whenever possible, frame your feedback as a curiosity.

❌ Toxic Phrasing ✅ Collaborative Phrasing
"This is inefficient." "Have we considered the performance impact here for large datasets?"
"Fix this function naming." "Would a more descriptive name like 'fetchUserData' help readability?"
"You forgot to handle the error." "What happens if the API returns a 500 here?"

The Power of "We"

Using "You" puts the recipient on the defensive. Using "We" makes it a team problem.

Bad: "You need to update the documentation."
Good: "We should probably update the docs so our future selves don't get confused."

The Sandwich Method (With a Twist)

The classic "Good, Bad, Good" sandwich is cliché. Instead, use the "Validation + Suggestion" model.

Example: "I really like how you handled the state management here; it's very clean. One suggestion: could we extract this logic into a utility function to make it reusable?"

This validates their effort first, making them more open to the suggestion.

Nitpicks vs. Blockers

Not all comments are equal. Explicitly label your comments to reduce anxiety:

  • [NIT] (Nitpick): "Minor indentation issue. Feel free to ignore if you're in a rush."
  • [BLOCKER]: "This logic introduces a security vulnerability. We need to fix this before merging."
  • [QUESTION]: "Just curious, why did we choose this library?"

Receiving Feedback Gracefully

When you are the one being reviewed, your job is to be "coachable."

If someone points out a bug, don't say: "Well, I did it that way because..." (Defensive).

Say: "Good catch. I hadn't considered that case. I'll fix it." (Professional).

Technical Jargon - Part 1

Master these essential terms and phrases for professional communication.

"Bandwidth"

Meaning: Capacity to handle work (not just network)

Example: I don't have the bandwidth to take on another project this sprint.

"Bottleneck"

Meaning: A point of congestion slowing down a process

Example: The database is the bottleneck in our system.

"Edge case"

Meaning: An unusual or extreme scenario

Example: We need to handle the edge case where the user has no internet.

"Tech debt"

Meaning: Accumulated shortcuts that need fixing later

Example: We have a lot of tech debt in the authentication module.

"Scalability"

Meaning: Ability to handle growth

Example: We need to improve the scalability of our API before launch.

"Latency"

Meaning: Delay in data transfer

Example: The latency between the server and client is too high.

"Throughput"

Meaning: Amount of work completed in a time period

Example: We increased throughput by 30% after optimizing the pipeline.

"Idempotent"

Meaning: Operation that can be repeated without changing the result

Example: The API endpoint should be idempotent to handle retries safely.

"Deprecate"

Meaning: To phase out old code/features

Example: We will deprecate the v1 API next quarter.

"Rollback"

Meaning: Reverting to a previous version

Example: We had to rollback the deployment due to a critical bug.

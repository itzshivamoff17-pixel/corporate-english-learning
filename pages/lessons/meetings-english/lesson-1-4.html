Executive Communication - Lesson 4

Reporting Progress to Leadership

Moving from "activities" to "outcomes" using military-grade communication standards.

The BLUF Standard (US Military)

The US Army manuals teach a communication standard called BLUF: Bottom Line Up Front. In a high-stakes environment, leaders don't have time to hunt for the point. The same applies to CTOs and VPs.

Anatomy of a BLUF Update

Start with the conclusion. Then provide supporting details.

❌ The "Mystery Novel" (Bad)

"So, we started looking at the logs on Monday, and we noticed some latency spikes. We then checked the database CPU usage, which was high. After investigating the slow query log, we found a missing index..."

(The leader is thinking: "Just tell me if it's fixed!")

✅ The BLUF (Good)

"BLUF: The latency issue is resolved. API response times are back to 50ms."

"Details: The root cause was a missing database index. We added it and are now monitoring..."

Reference: U.S. Army. (2015). Army Regulation 25-50: Preparing and Managing Correspondence.

The MECE Principle (McKinsey)

Consultants at McKinsey are trained to structure problems using the MECE principle: Mutually Exclusive, Collectively Exhaustive. When reporting progress, your updates should cover everything important without overlap.

Applying MECE to Weekly Reports

Don't just list random tasks. Group them into exhaustive categories:

  • 1. Delivered Value (The Past): What is now live for users?
  • 2. Work in Progress (The Present): What are we building right now?
  • 3. Risks & Blockers (The Future): What might stop us?

This structure ensures you haven't missed anything (exhaustiveness) and you aren't repeating yourself (exclusivity).

Reference: Rasiel, E. M. (1999). The McKinsey Way. McGraw-Hill.

Amazon's "Narrative" Culture

Amazon famously banned PowerPoint in executive meetings. Instead, they write 6-page memos (narratives). Why? Because "bullet points kill nuance."

Writing a Mini-Narrative Update

Instead of a bullet list of tasks, write a short paragraph connecting the dots.

Version A (Bullets):

  • Fixed bug #123
  • Updated React version
  • Met with design team

Version B (Narrative):

"This week, we focused on reducing technical debt to prepare for the Q4 traffic surge. By upgrading React and fixing the critical memory leak (bug #123), we've improved frontend stability by 20%. We also aligned with the design team to freeze UI assets by Friday."

Reference: Bryar, C., & Carr, B. (2021). Working Backwards: Insights, Stories, and Secrets from Inside Amazon. St. Martin's Press.

The "5-15" Report Format

Originally from Yvon Chouinard (Patagonia), this format is designed to be written in 15 minutes and read in 5 minutes.

1. Accomplishments Key wins from the week. Max 3 items.
2. Priorities Top 3 goals for next week.
3. Challenges Where do you need help/support?
4. Lesson Learned One insight to share with the team.
